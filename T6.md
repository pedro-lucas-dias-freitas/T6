## Introdução ao isomorfismo em árvores

Em muitos problemas com grafos, não basta saber quantos vértices e arestas uma
estrutura possui. Duas árvores podem ter o mesmo número de vértices, o mesmo
número de arestas e até a mesma sequência de graus, mas ainda assim terem
organizações internas diferentes. O problema de **isomorfismo em árvores**
consiste em decidir se duas árvores possuem exatamente a mesma estrutura, ainda
que os rótulos dos vértices sejam diferentes.

![alt text](imgs/exemplo.png)

**Figura 1. Exemplo de duas árvores isomorfas com rotulações diferentes.** A
figura ilustra que o problema não depende dos rótulos dos vértices, mas da
estrutura de adjacência preservada entre as duas árvores.

Neste trabalho, a comparação **não pode** ser feita por inspeção visual, por
“aparência” do desenho, por comparação informal de graus nem por assinaturas
heurísticas incompletas. O objetivo é implementar um método estrutural
correto: a **codificação canônica** de árvores.

A ideia central é transformar cada árvore em uma representação textual
canônica, independente da rotulação original dos vértices. Se duas árvores
produzirem a mesma codificação canônica, então elas são isomorfas; caso
contrário, não são.

Para que isso funcione, a árvore deve primeiro ser analisada a partir de seu
**centro**. Dependendo da estrutura, uma árvore pode ter um único centro ou
dois centros. A partir desse centro, a árvore é enraizada, suas subárvores são
codificadas recursivamente, os códigos dos filhos são ordenados
lexicograficamente e então concatenados entre parênteses. Esse processo gera
uma representação única para a estrutura da árvore.

Esse procedimento é especialmente importante porque evita decisões arbitrárias
dependentes da ordem de leitura do arquivo ou da numeração dos vértices. Uma
solução que funcione apenas em alguns exemplos ou que dependa da ordem de
inserção das arestas **não atende** ao objetivo deste trabalho.

### Exemplo simples de codificação canônica

Considere estas duas árvores enraizadas:

```text
Árvore 1:              Árvore 2:

    r                      x
   / \                    / \
  a   b                  y   z
     /                        \
    c                          w
```

As duas possuem a mesma estrutura:

- a raiz tem dois filhos;
- um filho é folha;
- o outro filho tem exatamente um filho folha.

Fazendo a codificação canônica:

- `a` é folha, então seu código é `()`;
- `c` é folha, então seu código é `()`;
- `b` possui um único filho folha, então seu código é `(())`;
- `r` possui filhos com códigos `()` e `(())`, portanto seu código final é
  `((())())`.

Na segunda árvore:

- `y` é folha, então seu código é `()`;
- `w` é folha, então seu código é `()`;
- `z` possui um único filho folha, então seu código é `(())`;
- `x` possui filhos com códigos `()` e `(())`, portanto seu código final
  também é `((())())`.

Como as duas árvores geram a mesma codificação canônica, conclui-se que elas
são isomorfas.

## Ideia do algoritmo canônico

O método exigido nesta atividade deve seguir a sequência conceitual abaixo:

1. verificar se cada entrada realmente representa uma árvore;
2. encontrar o(s) centro(s) por remoção iterativa de folhas;
3. enraizar a árvore em um centro;
4. codificar recursivamente cada subárvore;
5. ordenar os códigos dos filhos;
6. comparar as codificações canônicas finais.

No caso de uma árvore com dois centros, deve-se tratar corretamente essa
situação para obter uma representação canônica consistente. Em outras palavras,
uma implementação que assuma sempre centro único está incompleta.

## Algoritmo para encontrar o(s) centro(s) de uma árvore

**Entrada:** Árvore não direcionada `T = (V, E)`  
**Saída:** Conjunto com um ou dois centros

1. calcular o grau de cada vértice
2. colocar em `L` todos os vértices com grau `0` ou `1`
3. `processados <- |L|`
4. **enquanto** `processados < |V|` **faça**
   1. criar lista vazia `novasFolhas`
   2. para cada vértice `u` em `L`
      1. para cada vizinho `v` de `u`
         1. decrementar o grau de `v`
         2. se o grau de `v` se tornar `1`, inserir `v` em `novasFolhas`
   3. `processados <- processados + |novasFolhas|`
   4. `L <- novasFolhas`
5. devolver `L`

## Algoritmo de codificação canônica

**Entrada:** Árvore enraizada em `r`  
**Saída:** Código canônico da árvore

1. se `r` não possui filhos, devolver `"()"`
2. para cada filho `c` de `r`, calcular recursivamente `encode(c)`
3. ordenar lexicograficamente os códigos dos filhos
4. concatenar os códigos ordenados
5. devolver `"(" + concatenacao + ")"`

## Tema

Implemente, em **Python** ou em **Java**, um programa que leia **duas árvores
não direcionadas** em arquivos distintos no formato `algs4` e determine se
elas são isomorfas usando **codificação canônica**.

## Bases oficiais

As bases oficiais da disciplina para este trabalho são:

- `algs4-py`, para implementações em **Python**;
- `algs4-java`, para implementações em **Java**.

A estrutura principal de representação do grafo deve usar as classes padrão da
base adotada:

- `graph.py`, em Python;
- `Graph.java`, em Java.

Não atende ao trabalho substituir essa estrutura principal por bibliotecas
externas ou por outra representação de grafo como estrutura base da solução.
Em outras palavras, a árvore deve ser **lida, armazenada e percorrida
inicialmente** usando `Graph.java` ou `graph.py`.

É permitido criar uma **estrutura auxiliar interna** durante o algoritmo, como
uma árvore enraizada temporária ou nós auxiliares para a etapa de codificação
canônica. Porém, essa estrutura auxiliar **não substitui** a exigência de usar
`Graph.java` ou `graph.py` como representação principal da entrada.

## Entrada

O programa deve receber **dois caminhos** por linha de comando:

```bash
java Main arquivo1.txt arquivo2.txt
```

ou

```bash
python main.py arquivo1.txt arquivo2.txt
```

Cada arquivo deve seguir o padrão `algs4` para grafos não direcionados:

```text
V
E
v1 w1
v2 w2
...
```

onde:

- `V` é o número de vértices;
- `E` é o número de arestas;
- cada linha `v w` indica uma aresta não direcionada entre `v` e `w`.

Os vértices devem ser indexados de `0` até `V - 1`.

## O que o programa deve fazer

O programa principal deve:

1. receber dois arquivos de entrada por linha de comando;
2. construir os dois grafos usando `Graph.java` ou `graph.py`;
3. exibir a lista de adjacência de cada árvore;
4. validar se cada entrada representa uma árvore;
5. informar o(s) centro(s) de cada árvore válida;
6. informar a codificação canônica obtida para cada árvore válida;
7. informar o veredito final de isomorfismo.

Se alguma das entradas não representar uma árvore válida, o programa deve
interromper a comparação e informar claramente o motivo.

## Saída esperada

O programa deve exibir de forma clara, no mínimo:

- a lista de adjacência da primeira árvore;
- a lista de adjacência da segunda árvore;
- a validação de cada entrada;
- o(s) centro(s) encontrado(s) em cada árvore válida;
- a codificação canônica de cada árvore válida;
- o veredito final: isomorfas ou não isomorfas.
